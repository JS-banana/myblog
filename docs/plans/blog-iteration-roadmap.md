## 范围边界

- 首页定位不变：`首页=文章列表`，不承载“我是谁”介绍（介绍留在 `/about`）。
- 域名、部署、远端同步联通仍后置。
- 维持当前技术约束：Astro + 纯 CSS，不新增 UI 框架。

## Now（第 1 阶段：先把首页从骨架拉到可读内容流）

目标：保持极简，但让首页看起来是“有内容的站点”，不是“模板骨架”。

### 待完成

- [DONE] 列表项升级为：标题 + 日期 + 摘要（摘要优先 `description`，fallback 正文截断，120 字以内）。
- [DONE] 首页继续保持纯时间倒序流，不新增“精选/置顶”区。
- [DONE] 导航新增内容发现入口：归档（`/archives`）。
- [DONE] 用真实文章做渲染验证（13 篇，覆盖 2021/2022/2024/2026）。

### 验收标准

- 首页首屏在移动端/桌面端均可在 3 秒内读懂“最近内容是什么”。
- 列表项摘要不会导致布局错位（长文标题与短文标题都可读）。
- 至少 1 个“内容发现入口”可点击并返回合理页面。
- 使用真实文章验证，不以 `hello-world` 作为唯一验证内容。

## Next（第 2 阶段：补齐内容发现链路）

目标：让用户能快速找到“特定主题或历史文章”。

### 待完成

- 归档页（按年，必要时按月分组）。
- 标签页（聚合 tag，支持 tag 路由）。
- 搜索页（Pagefind 方案）：
  - 构建链路接入 `pagefind --site dist`
  - 搜索结果与 URL 参数 `q` 同步
- 列表分页体验优化（当前页定位、可访问性文案）。

### 验收标准

- 任意一篇文章都能通过“时间/标签/搜索”至少两种路径被定位到。
- `npm run build` 仍稳定通过，构建时长变化可接受（记录基线前后对比）。
- 新增页面在 SEO 头信息上不缺失 canonical / title / description。
